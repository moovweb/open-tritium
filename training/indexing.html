Indexing an Element

Index

The index() function is used to keep track of how many times a Tritium function iterates.

After a selector, the index() function will go through each possible selection of the element and assign a number based on its index. Each iteration will increase the index by 1. So the first element will be “1”, the second “2” and so on.

The index could then be used

  • in a log statement:

    $("//div") {
  log(index())
}

    will iterate through every <div> in the document and log a number per <div> — the last log message will be the total number of <div>s in the structure.

  • to add a class to elements:

    $("//table") {
  add_class(index())
}

    will take every <table> and add a class based on where the <table> is in the DOM — the first <table> will have the class "1", and so on.

Note that because it counts iterations of how many times the previous function is performed, the index() function will not assign an index value based on sibling number (see example below).

Example

Given the following HTML:

<ul>
  <li></li>
  <li></li>
  <li></li>
  <li></li>
  <li></li>
</ul>

We can use the index() function to assign different <li>s unique classes.

The following Tritium:

$("./ul") {
  $("./li") {
    attribute("class", index())
  }
}

will give the following output:

<ul>
  <li class="1"></li>
  <li class="2"></li>
  <li class="3"></li>
  <li class="4"></li>
  <li class="5"></li>
</ul>

The following Tritium:

$("./ul") {
  $("./li[3]") {
    attribute("class", index())
  }
}

will give the following output:

<ul>
  <li></li>
  <li></li>
  <li class="1"></li>
  <li></li>
  <li></li>
</ul>
